## 5. Executive Dashboard

**Verzeichnis:** `dashboard/`

### Backend — `dashboard/backend/server.py`
- FastAPI + WebSocket auf Port 8081
- EventBus: Ring-Buffer mit 500 Events
- Status-Poller alle 5s, Agent-Health alle 30s
- Instrumentierter Pipeline-Runner — emittiert Live-Events
- REST: `/api/status`, `/api/events`, `/api/agents`, `/api/health`

### Frontend — `dashboard/frontend/` (React + Vite)
- **TopBar:** Equity, Drawdown, Pipeline-Status, Run/Halt/Resume
- **PipelineFlow:** 8 Agent-Nodes — aktiver Agent leuchtet blau, Kill-Stage rot
- **TradeFeed:** Scrollendes Live-Log (Trades grün, Kills rot, Signals blau)
- **DrawdownMeter:** Animierter Balken + Circuit-Breaker-Grid
- **ReasoningPanel:** Typewriter-Effekt für ZEUS LLM-Reasoning
- **EquityChart:** Recharts Area Chart aus WebSocket Status-Events

Umgestellt von `react-scripts` auf **Vite** (react-scripts inkompatibel mit Node 24).

Start lokal:
```
Backend:  python -m uvicorn dashboard.backend.server:app --port 8081 --reload
Frontend: cd dashboard/frontend && npm start  →  http://localhost:3001
```

## 6. Cloud-Infrastruktur

**Entscheidung:** Hetzner CX21 + Supabase Pro + Cloudflare Pages
**Begründung:** Railway hat EU-Outage-Geschichte, Hetzner läuft 24/7 ohne Cold Starts.
Supabase gibt uns Postgres + pgvector + Realtime WebSocket in einem Service.

### Supabase Schema — `infra/supabase/001_schema.sql`

11 Tabellen:
- `signals` — Icarus-Output (Raw Signals)
- `filtered_signals` — Hades-Output (Compliance)
- `macro_context` — Artemis-Output (Regime Snapshots)
- `trades` — ersetzt SQLite `trade_log.db`
- `decision_traces` — vollständiger Audit Trail (ersetzt ChromaDB decisions)
- `portfolio_state` — Argus Equity Snapshots (Time-Series)
- `portfolio_positions` — offene Positionen
- `agent_health` — Watchdog Reports
- `circuit_breaker_state` — CB Snapshots
- `knowledge_documents` — ersetzt ChromaDB komplett (pgvector, HNSW Index)
- `ticker_map` — ersetzt `data/ticker_map.json`

Features:
- pgvector mit HNSW Index für Vektor-Suche
- Materialized View `trade_hit_rates` für Pythia
- Supabase Realtime auf 4 Tabellen aktiviert (Trades, Traces, Portfolio, Health)
- Row Level Security: Service Role = Full Access, Anon = Read-Only

`infra/supabase/002_seed_ticker_map.sql` — 43 vorgemappte Supplier (XETRA/NYSE/NASDAQ)

### Hetzner Setup — `infra/hetzner/`
- `docker-compose.prod.yml` — ZEUS + Dashboard + Redis + Nginx
- `nginx.prod.conf` — SSL-Terminierung, WebSocket-Proxy
- `setup.sh` — One-Shot Bootstrap Ubuntu 24.04 (Docker, UFW, Certbot, systemd)

### Cloudflare Pages — `infra/cloudflare/`
- `wrangler.toml` — Projekt-Konfiguration
- `_redirects` — SPA Fallback + API Proxy

### CI/CD — `.github/workflows/deploy.yml`
Push auf main → automatisch:
1. Docker Image builden → GHCR pushen
2. SSH zu Hetzner → `docker compose pull && up`
3. React builden → Cloudflare Pages deployen

### Supabase Realtime Hook — `dashboard/frontend/src/hooks/useSupabaseRealtime.js`
React subscribed direkt auf Postgres-Änderungen via WebSocket.
Jeder neue Trade erscheint sofort im Dashboard ohne Polling.

## 8. MilestoneManager — Capital Preservation

**Datei:** `core/milestone_manager.py`

### Das Eiserne Gesetz
> Nie alles riskieren. Immer die Basis schützen. 30% der Gewinne → Vault. Permanent.

### 5 Stages

| Stage | Kapital | Max Position | Kill Switch | Leverage |
|-------|---------|-------------|-------------|----------|
| 🌱 SEED | €100 → €1.000 | 1% | 5% DD | 1.0x |
| 🔥 SPRINT | €1.000 → €10.000 | 2% | 6% DD | 1.0x |
| 💎 SCALE | €10.000 → €100.000 | 3% | 7% DD | 1.5x Tier-1 |
| 🏦 SERIOUS | €100.000 → €1.000.000 | 5% | 8% DD | 2.0x Tier-1 |
| 🚀 EMPIRE | €1.000.000+ | 3% | 5% DD | 2.0x Tier-1 |

### Vault-Logik
- Bei jedem Milestone-Crossing: Telegram-Alert mit Betrag
- Manuelles Bestätigen via `/confirm_vault`
- ZEUS hat niemals Zugriff auf den Vault

### Integriert in:
- **Pythia:** Hard Cap auf `max_position_pct` pro Stage, Tier-Blocking
- **Argus:** Kill-Switch-Schwelle aus Milestone-Config
- **ZEUS:** `ZeusConfig.starting_equity = 100.0`, `get_milestone_status()` API
