### Task 6 — Kafka Event Bus (Pre-Live Infrastructure Hardening)

**Why**: The current pipeline is synchronous and polling-based — n8n triggers ZEUS every 15 minutes, all stages run in sequence, and if a signal is missed in a cycle it is gone. Kafka solves three concrete risks before they become live trading incidents:

1. **Signal loss** — two high-urgency signals arriving in the same 15-minute window: only one gets processed. With Kafka, both are persisted on the bus and consumed in order.
2. **No replay** — if ZEUS has a bug and rejects 3 hours of valid signals, those signals cannot be recovered. Kafka retains the full signal stream for a configurable retention window (default: 7 days).
3. **Pipeline blocking** — if one stage hangs (IBKR slow, ChromaDB slow), the whole synchronous pipeline stalls. Kafka decouples the stages: Icarus publishes to a topic, ZEUS consumes independently. One stage's slowness does not block the others.

**Architecture change**:
```
Before (synchronous):
  n8n → ZEUS._process_signal(raw_signal) → Icarus → Hades → Artemis → Pythia → Ares → Argus

After (event-driven):
  Hermes → Icarus → Kafka topic: raw_signals
                          ↓
                    ZEUS consumer → Hades → Artemis → Pythia → Ares → Argus
                          ↓
                    Kafka topic: decision_traces (for Apollo replay / audit)
```

**Implementation steps**:
1. Add Kafka (KRaft mode — no Zookeeper) to `infra/hetzner/docker-compose.prod.yml` as a new container
2. Create two topics: `zeus.raw_signals` (retention: 7 days), `zeus.decision_traces` (retention: 30 days)
3. Refactor Icarus: instead of returning `RawSignal` directly, publish to `zeus.raw_signals`
4. Refactor ZEUS: replace direct Icarus call with a Kafka consumer loop (one message per pipeline run)
5. After each `DecisionTrace` is written to ChromaDB, also publish it to `zeus.decision_traces` for Apollo replay
6. Add `kafka-python` or `confluent-kafka` to `requirements.txt`
7. Update `tests/conftest.py` to mock the Kafka producer/consumer so tests remain offline-capable

**Docker service to add** (`docker-compose.prod.yml`):
```yaml
kafka:
  image: apache/kafka:3.7.0
  environment:
    KAFKA_NODE_ID: 1
    KAFKA_PROCESS_ROLES: broker,controller
    KAFKA_LISTENERS: PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093
    KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:9092
    KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka:9093
    KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
    KAFKA_LOG_RETENTION_HOURS: 168
  volumes:
    - kafka_data:/var/lib/kafka/data
```

**Sequencing**: complete after Task 1 (IBKR paper trading connected) and before Task 4 (live execution). Do not go live on real money without the replay and decoupling guarantees Kafka provides.
